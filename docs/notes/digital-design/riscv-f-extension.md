# RISC-V F 扩展：单精度浮点状态与指令

F 扩展为 RISC-V 增加 IEEE 754 兼容的单精度浮点运算、32 个浮点寄存器和浮点控制状态。它依赖 `Zicsr`，因此浮点状态通过 CSR 暴露给软件；是否允许执行浮点指令还受特权状态中的 `FS` 字段影响。

## 寄存器与 `fcsr`

F 扩展增加 `f0`–`f31`，浮点寄存器宽度 `FLEN=32`。整数寄存器与浮点寄存器是不同的寄存器文件：算术结果不会自动跨文件移动。

`fcsr` 的低 8 位由两部分组成：

| 字段 | 位 | 作用 |
| --- | --- | --- |
| `fflags` | `[4:0]` | 粘滞异常标志：NV、DZ、OF、UF、NX |
| `frm` | `[7:5]` | 动态舍入模式 |

`fflags` 记录无效操作、除零、上溢、下溢和非精确；它们不是传统意义上的逐条同步 trap。软件若关心状态，应在适当时机读取并清除标志。舍入模式可由指令的 `rm` 字段指定，也可选择动态模式并读取 `frm`。

## 指令分组

| 分组 | 代表指令 | 说明 |
| --- | --- | --- |
| 算术 | `fadd.s`、`fsub.s`、`fmul.s`、`fdiv.s`、`fsqrt.s` | 单精度运算 |
| 融合乘加 | `fmadd.s`、`fmsub.s`、`fnmsub.s`、`fnmadd.s` | 单次舍入的乘加族 |
| 符号/比较 | `fsgnj.s`、`fmin.s`、`fmax.s`、`feq.s`、`flt.s`、`fle.s` | 比较结果写入整数寄存器 |
| 加载/存储 | `flw`、`fsw` | 在内存与浮点寄存器间传输单精度值 |
| 转换/位移动 | `fcvt.*`、`fmv.x.w`、`fmv.w.x` | 数值转换或保持位模式的跨寄存器文件移动 |
| 分类 | `fclass.s` | 返回 NaN、无穷、正规数、非正规数、零等分类位 |

`fsgnj.s` 可用于表达符号复制；汇编器常把绝对值或取负写成伪指令，再展开为 `fsgnjx.s` / `fsgnjn.s` 一类基本指令。写 RTL 解码或检查 ISA 覆盖时，应以真实编码的基本指令为准。

```riscv
# fa0 = fa0 * fa1 + fa2；只有一次浮点舍入
fmadd.s fa0, fa0, fa1, fa2

# 位模式移动，不执行数值转换
fmv.x.w t0, fa0
fmv.w.x fa1, t0

# 数值转换，受 rm/frm 影响
fcvt.w.s a0, fa0, rtz
```

## 特权与上下文切换

`mstatus.FS`（以及存在 S-mode 时的相应状态）管理浮点上下文。`FS=Off` 时执行浮点指令会触发非法指令异常；状态从 Initial/Clean 变为 Dirty 后，操作系统通常需要在上下文切换时保存浮点寄存器与 `fcsr`。

这也是延迟保存（lazy save）策略的依据：从未使用浮点的任务不必承担 32 个寄存器的保存成本。但中断、信号、虚拟化和调试器都必须遵守同一套保存规则，不能只在普通任务切换路径处理。

## 实现与验证检查点

- 区分 `FMV` 的位拷贝和 `FCVT` 的数值转换；二者的 NaN 与舍入行为不同。
- 对每种舍入模式、NaN、无穷、次正规数、零和溢出建立参考模型测试。
- 检查 `fflags` 是粘滞置位而非被无关运算清零；验证显式清除路径。
- 在支持 D/Q 或向量浮点时，确认 NaN-boxing、`FS`/`VS` 状态及不同 `FLEN` 的交互。
- 非对齐 `flw/fsw` 的结果取决于执行环境：可能由硬件处理，也可能发生异常，软件不能假定所有实现相同。
